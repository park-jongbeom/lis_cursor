# 구현 · 사용자 확인 · 테스트 게이트 — 필수 워크플로

> **목적**: 구현이 끝난 직후 곧바로 다음 Phase 문서로 넘어가거나, 테스트 없이 마감하는 일을 방지한다.  
> **적용 대상**: Cursor AI 및 모든 기여자.  
> **연계 문서**: `.cursorrules` 작업 문서 체계, `docs/rules/project_context.md` §3단계 작업 프로토콜, `docs/CURRENT_WORK_SESSION.md`.

---

## 요약 다이어그램

```text
[계획/체크리스트] → [구현] → [CURRENT에 구현 완료 기록] → ⏸ 사용자 확인
                                                      ↓
                                        [테스트 계획 수립·기록] → ⏸ 사용자 승인
                                                      ↓
                                        [테스트 작성·실행·검증 기록] → ⏸ 사용자 확인(선택)
                                                      ↓
                            WORK_HISTORY 이전 + CURRENT를 다음 세션 과제로 교체
```

---

## 규칙 (준수 순서)

### Gate A — 구현 완료 시 (코딩 종료 직후)

1. **`docs/CURRENT_WORK_SESSION.md`를 즉시 다음 세션 주제로 덮어쓰지 않는다.**
2. 동일 세션 문서 안에 **「구현 완료 요약」**을 작성한다.
   - 변경·추가된 파일 경로, 주요 설계 결정, 알려진 제한 사항.
3. 체크리스트에서 완료된 항목은 `- [x]`로 표시한다.
4. **진행 상태**를 `구현 완료 — 사용자 확인 대기`로 바꾼다.
5. 사용자에게 **구현 내용 검토를 요청**하고, **명시적 확인(승인) 전에는 테스트 계획(Gate B)으로 넘어가지 않는다.**

### Gate B — 테스트 계획 (사용자 확인 후)

1. 사용자가 구현을 확인한 뒤에만 **「테스트 계획」** 섹션을 채운다.
   - 범위: 단위 / 통합 / 수동 시나리오 중 무엇을 할지, 우선순위, 사용할 명령(예: `pytest` 경로, `PYTHONPATH`).
2. 진행 상태를 `테스트 계획 수립 완료 — 실행 승인 대기`로 둘 수 있다(팀 관행에 맞게).
3. 사용자 승인 후에만 테스트 코드 작성 및 실행에 들어간다.

### Gate C — 테스트 검증

1. 테스트 실행 결과(통과/실패, 커버리지 요약이 있으면 함께)를 **「테스트 검증 결과」**에 기록한다.
2. 실패 시 수정 루프를 거친 뒤 다시 실행하고, 결과를 갱신한다.
3. **전부 통과**하고 사용자가 최종 확인하면:
   - 상세 이력을 `docs/history/WORK_HISTORY.md`에 옮기고,
   - `CURRENT_WORK_SESSION.md`를 **다음 세션(다음 Phase) 과제만** 담도록 갱신한다(완료된 구현/테스트 장문은 WORK_HISTORY에만 둔다).

---

## AI(Cursor) 동작 금지 사항

- 사용자 확인 없이 **Session N+1 문서로 `CURRENT_WORK_SESSION.md` 전체 교체**하기.
- Gate C 없이 **`WORK_HISTORY`에만 올리고 세션을 넘기기**하기(검증 결과가 문서에 없으면 안 됨).
- 프로젝트에 **테스트가 정하기로 한 범위에 포함되는데**, 승인 없이 테스트를 생략하기.

## `CURRENT_WORK_SESSION.md` 표준 섹션 (권장 순서)

1. 세션 메타(Phase, plan.md 참조, **진행 상태**, 완료 기준)
2. 환경·전제
3. **세션 워크플로 상태** (표: 구현 / 사용자 확인 / 테스트 계획 / 테스트 검증)
4. 구현 체크리스트(기존 Phase 단계)
5. **구현 완료 요약** (Gate A에서 채움)
6. **테스트 계획** (Gate B)
7. **테스트 검증 결과** (Gate C)
8. 완료 후 처리(체크리스트, WORK_HISTORY 이전, 다음 세션 예고)

세션이 아직 시작 전이면 5~7은 비워 두거나 `— 진행 시 작성 —`으로 둔다.
